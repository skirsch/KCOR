## Repo-specific habits

- Prefer WSL for `make` targets and Monte Carlo runs; several recipes assume Unix tools such as `rm`, `mv`, `ls`, and Linux `fork`.
- On Windows shells, direct Python invocations can work for small tasks, but use `PYTHONIOENCODING=utf-8` if output contains non-ASCII characters.
- For Monte Carlo KCOR runs, distinguish iteration `0` from bootstrap draws:
  - iteration `0` is the original-data point estimate;
  - later iterations are bootstrap resamples.
- For percentile bootstrap CIs, report the point estimate from iteration `0` and compute CI percentiles from bootstrap draws only unless explicitly stated otherwise.
- KCOR birth-year rows are bucketed by `AGE_RANGE`; with the default `AGE_RANGE=10`, raw birth years `1940-1949` appear as output row `YearOfBirth=1940`.
- When changing executable behavior or environment variables, update `docs/executables.md`.
- After meaningful KCOR runs, append a concise entry to `process/work_log.md` with command, outputs, and headline result.
