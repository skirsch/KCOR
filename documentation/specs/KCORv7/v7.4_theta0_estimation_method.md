# Theta0 Estimation Under Epidemic Waves: The Delta-Iteration Method

## The Problem

The KCOR estimator fits a gamma-frailty Gompertz model to observed cohort hazard data to recover
`theta0` — the frailty variance at enrollment. The marginal cohort hazard is:

$$h_\text{obs}(t) = \frac{k \cdot e^{\gamma t}}{1 + \theta_0 \cdot H_0^\text{eff}(t)}$$

where `H0_eff(t)` is the cumulative individual-level baseline hazard. During quiet periods,
`H0_eff(t) = H_gom(t)` — the pure Gompertz cumulative hazard. But during an epidemic wave,
`H0_eff` accumulates **faster** than `H_gom` because individuals face elevated mortality.

After the wave ends, `H0_eff(t)` carries a **permanent excess** over `H_gom(t)`:

$$H_0^\text{eff}(t) = H_\text{gom}(t) + \delta \quad \text{for all } t \geq t_\text{wave end}$$

The wave has **fast-forwarded** the population's frailty depletion clock. The surviving cohort
is systematically healthier than the pure Gompertz formula predicts.

### The Original Bug

The original estimator fitted the same formula to all quiet windows — pre-wave and post-wave —
using the same time index `t` and the same `H_gom(t)`. But post-wave quiet points sit on a
**different branch** of the curve, shifted forward by `delta`. The optimizer compensated by
inflating `theta_hat`, producing **+14% bias** on post-wave windows.

This is equivalent to having a smooth curve cut into segments with unknown gaps, then trying to
fit a single curve through all segments without accounting for the gaps. The fit fails because
the segments are inconsistent with any single `theta`.

---

## The Key Insight: Waves Are Time Gaps

An epidemic wave does not change the **shape** of the Gompertz-frailty curve. It only
**fast-forwards** the population along it by depositing extra cumulative hazard `delta`.

Therefore:
- All quiet windows — pre and post any number of waves — lie on the **same smooth curve**
- The wave creates a **jump** in `H0_eff` that can be measured from the data
- Once each jump `delta_i` is known, all quiet windows are internally consistent
- A single `theta` fits all of them perfectly

This is the missing link: **treat each wave gap as a single scalar unknown, computed directly
from the data, not fitted.**

---

## The Delta-Iteration Algorithm

### Parameters
- `k` — Gompertz baseline hazard at `t=0`
- `theta0` — frailty variance at enrollment
- `delta_i` — excess cumulative hazard deposited by wave `i` (one per wave)

### Step 1: Read k Exactly

At `t=0`, `H_gom(0) = 0`, so the denominator equals 1:

$$h_\text{obs}(0) = k$$

`k` is read directly from the data — **no fitting required, no approximation**.

### Step 2: Initial theta from First Quiet Window

Fit a single-parameter nonlinear least squares on the pre-wave quiet window:

$$\hat{\theta}_0 = \arg\min_\theta \sum_{t \in W_0} \left[ h_\text{obs}(t) - \frac{k \cdot e^{\gamma t}}{1 + \theta \cdot H_\text{gom}(t)} \right]^2$$

This gives an approximate `theta` (~1% error) to seed the iteration.

### Step 3: Iterate to Convergence

Repeat the following until `theta` converges (typically 2–3 iterations):

#### 3a: Reconstruct H0_eff from h_obs

Using the gamma-frailty inversion, reconstruct the individual-level cumulative hazard
from the observed cohort hazard:

$$h_0^\text{eff}(t) = h_\text{obs}(t) \cdot \left(1 + \theta \cdot H_0^\text{eff}(t)\right)$$
$$H_0^\text{eff}(t+1) = H_0^\text{eff}(t) + h_0^\text{eff}(t)$$

starting from `H0_eff(0) = 0`. This reconstruction passes through **all weeks** —
quiet and wave alike — with no special cases. The wave weeks are handled automatically.

#### 3b: Compute Each Wave's Delta

For each wave `i` with known end time `t_i`:

$$\delta_i = H_0^\text{eff}(t_i) - H_\text{gom}(t_i)$$

This is a **direct subtraction** — not a fit. `delta_i` measures how much extra depletion
wave `i` deposited relative to the quiet Gompertz baseline.

For subsequent waves, deltas accumulate:

$$\Delta(t) = \sum_{i : t_i \leq t} \delta_i$$

#### 3c: Refit theta on ALL Quiet Windows

With `delta_i` values known, every quiet window is now on the same curve. Fit a single
parameter `theta` to all quiet windows jointly:

$$\hat{\theta} = \arg\min_\theta \sum_{t \in \bigcup_j W_j} \left[ h_\text{obs}(t) - \frac{k \cdot e^{\gamma t}}{1 + \theta \cdot (H_\text{gom}(t) + \Delta(t))} \right]^2$$

where `W_j` are all quiet windows and `Delta(t)` is the accumulated delta up to time `t`.

---

## Properties

| Property | Value |
|---|---|
| Free parameters | 1 (`theta` only) |
| Parameters read from data | 1 (`k` from `h_obs(0)`) |
| Parameters computed analytically | N (`delta_i` per wave) |
| Iterations to convergence | 2–3 |
| Bias with perfect data | < 0.01% |
| Requires wave multiplier knowledge | No |
| Requires wave timing knowledge | Yes (to read delta_i) |
| Handles N waves | Yes — one delta per wave |

## Generalization to N Waves

For a study period with N epidemic waves at known calendar intervals `[ws_i, we_i]`:

1. `k = h_obs(0)` — exact
2. Quiet windows: all time outside `[ws_i, we_i]` for all i
3. One `delta_i` per wave, read from the reconstructed `H0_eff`
4. Single `theta` fit to all quiet windows with accumulated deltas applied

The system is heavily over-determined: hundreds of quiet-window data points constrain
a single scalar `theta`. The more quiet windows, the more precise the estimate.

---

## Why the Original Method Was Biased

The original KCOR estimator used `H_gom(t)` for all windows including post-wave quiet windows.
Post-wave quiet points have `H0_eff(t) = H_gom(t) + delta`, so the denominator is **too small**
in the model, making the predicted hazard **too high**. The optimizer inflates `theta` to bring
the predicted curve down to match the data — producing the observed upward bias.

The delta-iteration method corrects this by using the **true** `H0_eff(t)` for each window,
which exactly accounts for all wave-accelerated depletion. The result is an unbiased, consistent
estimator that is exact in the limit of noiseless data.

## Limitations and Interpretation of Delta in Real Data

*This section addresses a valid critique: in real-world application, delta is not a pure
"wave effect" term.*

### What delta actually absorbs

In the synthetic simulation, delta has a clean interpretation: it is exactly the excess
cumulative hazard deposited by the epidemic wave, computable analytically. In real registry
data, delta absorbs **any deviation** between the true `H0_eff(t)` and the assumed
`H_gom(t)` over the non-quiet interval. This includes:

1. **Wave effects** — the intended signal: frailty-accelerated depletion from epidemic
   mortality. This is the dominant contribution in a well-specified study period.

2. **Model misspecification** — the Gompertz baseline `k·exp(γt)` is an approximation.
   If the true background hazard deviates from Gompertz (e.g. seasonal variation, cohort
   aging effects, changes in healthcare), `H_gom` will drift from `H0_eff` even during
   nominally quiet periods. Delta will absorb this drift.

3. **Baseline drift** — slow secular trends in mortality (improving treatments,
   changing comorbidity prevalence) can shift `k` or `γ` over multi-year follow-up.
   If the model uses fixed `k` and `γ`, the residual accumulates into delta.

4. **Misclassified wave boundaries** — if the quiet window boundaries are drawn
   incorrectly (wave starts earlier or ends later than specified), some wave mortality
   leaks into the quiet windows and contaminates the theta fit.

### Why this is acceptable, not disqualifying

Delta's role is to **bridge the gap** between pre-wave and post-wave quiet segments so
that theta can be estimated consistently from all quiet windows. Whether that gap is
100% wave effect or 90% wave + 10% model drift, the theta estimate is still the best
achievable given the model assumptions. The key guarantee is:

> *If the Gompertz-frailty model is correctly specified during quiet windows, theta is
> identified exactly regardless of what happened between those windows.*

Delta is a **nuisance parameter** in the statistical sense — it is not of primary
interest, it is estimated to remove confounding, and its interpretation is secondary
to the theta estimate it enables.

### Practical implications

- **Delta should be reported** alongside theta in any real-data application, along with
  the wave period it corresponds to, so readers can assess whether the magnitude is
  plausible given the known wave severity.

- **Implausibly large delta** (e.g. larger than the entire pre-wave `H_gom`) is a
  diagnostic flag for model misspecification or incorrect wave boundary placement.

- **Delta cannot be negative** by construction (waves only accelerate depletion, never
  reverse it). A negative delta estimate is a sign of a misspecified model or
  miscategorized quiet window.

- **Multiple deltas across waves** provide an internal consistency check: if the model
  is well-specified, the ratio of delta_i to wave duration and multiplier should be
  roughly consistent across waves of similar severity.

### Summary

Delta is best described as:

> *The cumulative deviation of the true individual-level hazard from the assumed
> Gompertz baseline, integrated over the non-quiet interval. In a well-specified model
> with clearly identified epidemic waves, this is dominated by wave-accelerated frailty
> depletion. In general, it is a catch-all correction term that ensures pre-wave and
> post-wave quiet windows are mutually consistent under the fitted theta.*

This is not a weakness of the method — it is an honest acknowledgment of what can and
cannot be inferred from observational mortality data without individual-level covariates.

## Identifiability: When Does the Estimator Work Well?

Empirical sweep across 400 parameter combinations (20 mortality rates × 20 theta values,
two epidemic waves, 104-week follow-up) reveals a clear identifiability boundary.

### Results summary

| Follow-up | Worst-case error | Problem region |
|---|---|---|
| 2 years (104 weeks) | 4.93% | mortality < 3%/yr AND theta > 10 |
| 3 years (156 weeks) | 3.49% | mortality < 2%/yr AND theta > 12 |
| 4 years (208 weeks) | 2.14% | mortality < 2%/yr AND theta > 15 |

Outside the problem corner, errors are consistently **< 0.5%** regardless of follow-up.

### Root cause: geometric curvature

The estimator requires the h(t) curve to have enough **curvature** over the observation
window for the optimizer to distinguish theta from k. The total curvature is governed by:

```
curvature ∝ annual_mortality × n_years × theta × gamma
```

When this product is small — low mortality, short follow-up, small theta — the h(t) curve
is nearly flat and k and theta become nearly collinear. More iterations do not help: the
algorithm converges to a fixed point in 2 iterations, but that fixed point is not the true
theta because the data is geometrically insufficient to identify it.

### Practical implication

For typical all-cause mortality cohorts (≥ 4%/yr) with theta in the range 1–10 and
2+ years of follow-up, the estimator is well-identified with errors well under 1%.
The Czech Republic registry data (≈6%/yr, 2+ year follow-up) falls comfortably in
the well-identified zone.

For low-mortality cohorts or very short follow-up, the identifiability limit is a
fundamental geometric property of the model — not a limitation of the algorithm —
and cannot be overcome by any estimator without additional assumptions.
